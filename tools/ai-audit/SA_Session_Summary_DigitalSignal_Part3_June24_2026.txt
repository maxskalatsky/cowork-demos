SECTION C — DUAL-MODE SCORING (bf01a43)
─────────────────────────────────────────────────────────────

ARCHITECTURE:

Every submitted URL routes through classifyEntity() before any scoring runs. This is a
Claude Haiku call (max_tokens=10) that returns exactly "private_company" or "known_brand".
On any API failure or missing key it defaults to "private_company" — the existing pipeline
is never broken by a classification error.

classifyEntity() is called after the KV email gate (repeat emails are blocked before the
expensive classification call runs) and before the main try block.

Known brand definition passed to Claude:
  publicly traded company, Fortune 1000, major media or sports property,
  government entity, utility, or globally recognized consumer brand.
  Everything else = private_company.

PRIVATE COMPANY PATH: Standard pipeline — DataForSEO + agent signals fetch, SEO scoring,
positioning, agent readiness, four findings, Forward Signal, Compare mechanic.

ENTERPRISE PATH (known_brand):
1. fetchAgentSignals() — crawl with Scrapfly/Puppeteer fallback
2. dataForSeoOnPage() — wrapped in try/catch, falls back to empty {checks:{},meta:{}}
3. scoreEnterprise() — four dimensions, each tier 1–5
4. KV log — records entityType:"known_brand" instead of SEO score
5. getEnterpriseBenchmarkSignal() — Sonnet 4.6, three sentences, benchmark framing
6. Response: {entityType:"known_brand", brandAuthority, audienceClarity, aiVisibility,
   contentDepth, forwardSignal}

ENTERPRISE DIMENSION SCORING (scorer.js, scoreEnterprise()):

Brand Authority: starts at 8 (Dominant baseline), deducts for failures.
  No JSON-LD:      -2
  Not HTTPS:       -4
  No canonical:    -1
  No description:  -1
  Tier: >=7→1, >=5→2, >=3→3, >=1→4, else→5

Audience Clarity: additive from 0.
  Link count >=30: +3, >=15: +2, >=7: +1
  h2 count >=8: +3, >=4: +2, >=2: +1
  Content-type keyword diversity (news/video/live/fantasy/scores/sports/podcast/etc.) up to +3
  All three heading levels present: +1
  Tier: >=10→1, >=6→2, >=3→3, >=1→4, else→5

AI Visibility: additive from 0.
  llms.txt present: +3
  AI crawlers not blocked: +2
  JSON-LD: +2
  FAQ/Article/HowTo schema: +1
  H1+H2 both present: +1
  Word count >=300: +1
  Tier: >=8→1, >=6→2, >=4→3, >=2→4, else→5

Content Depth: additive from 0.
  /news|blog|stories|articles section path in HTML: +2
  /video|videos|watch section path: +1
  /scores|standings|results section path: +1
  /insights|resources|learn|knowledge|research section path: +1
  Link count >=50: +2, >=20: +1
  h2 count >=6: +2, >=3: +1
  Word count >=1000: +1
  Tier: >=7→1, >=4→2, >=2→3, >=1→4, else→5

SCORING CALIBRATION RATIONALE:

Brand Authority starts from 8 because additive scoring from zero requires the auditor to
observe rare signals (sameAs, structured data with specific properties) that many Fortune
500 brands would not reliably trigger. A functioning known brand should be Dominant by
default — deducting for hard failures is more honest than rewarding for rare signals.

Audience Clarity uses content-type diversity (news, video, live, sports, etc.) instead of
B2B-only audience pattern matching ("for enterprise|teams|individuals|developers"). ESPN
and similar consumer media brands have none of that B2B vocabulary. Link count + h2 breadth
+ content-type diversity measures segmentation directly without category bias.

Content Depth uses section path detection in HTML links (/news, /video, /scores) instead
of home-page word count. ESPN's home page shell is thin, but its section paths reveal deep
content. Section paths present in href links are a reliable signal regardless of what the
home page renders.

SECTION F — THREE-QUESTION INTAKE FORM
─────────────────────────────────────────────────────────────

Added a three-question intake section between the URL input and Submit button in
digital-audit.html. Questions are placed inside a horizontal rule divider and styled

QUESTION 1 — What best describes this company? (radio, required)

  Large national or global brand     → entityType: "large_national_global"

  Existing clients or customers      → primaryVisitor: "existing_clients"

  Serve existing relationships       → siteGoal: "serve_relationships"
  Inform or educate                  → siteGoal: "inform_educate"

Values are serialized as entityType (string), primaryVisitor (array), siteGoal (array)
in the request body JSON.

CUSTOM INDICATORS:
Radio buttons use ::before pseudo-element with radial-gradient for the selected state:
  unselected: white circle with light border (#c8d4de)
  selected: green circle (#1F6B4E) with white center dot via radial-gradient(circle, #fff 28%, #1F6B4E 28%)

Checkboxes use ::before pseudo-element with SVG data URI for the checkmark:
  unselected: white square with light border, 3px radius
  selected: teal square (#4A9E8F) with white SVG checkmark via data URI background

Both use :has() pseudo-class to detect checked state without sibling combinators.
Both are visible at all times — no display:none on the native input elements (just
opacity:0 so the native input is hidden behind the custom indicator).

VALIDATION:
The Submit button is disabled by default. intakeValid() checks all five conditions.
updateBtn() is called on change events for all three intake groups and on input events
for the email and URL fields. stopLoading() calls updateBtn() instead of unconditionally

CHANGE 1 — CLEAN LETTER GRADES:

throughout, so no front-end changes were needed for the compare path.

CHANGE 2 — CALIBRATION APPLIED IN SCORING:

seoBaselineGrade override (buildRulesReport):
  If raw SEO grade < profile baseline AND hard failure count < 2:
    grade is lifted to the profile baseline.
  Hard failures: missing canonical, not HTTPS, broken page, dom_complete > 5000ms.
  Applied before the grade is returned from buildRulesReport — report.seoGrade reflects
  the calibrated grade, not the raw score-derived grade.

authorityPenaltyReduced (buildRulesReport):
  If profile.calibration.authorityPenaltyReduced is true AND raw authority band is "critical":
    authority band is treated as "weak" instead.
  Affects: findings tile label, text, and border color for the Authority tile.
  Rationale: SMBs and nonprofits are not expected to have competitive backlink profiles.
  A "critical" authority label for a local law firm is misleading and demoralizing.

hookFailureIsCritical (worker.js):
  If profile.calibration.hookFailureIsCritical is false AND positioning.hook === "missing"
  AND positioning.color === "red":
    positioning.color is overridden to "yellow".
  The hook value and verdict text are not modified — the hook is still "missing" in the
  data, just not treated as a card-level critical failure.
  Rationale: A missing or generic headline is not a conversion crisis for an established
  firm serving existing relationships or building authority. It matters most for SMBs
  doing cold outreach.

Dimension weights in Forward Signal (signals.js + worker.js):
  The resolved profile's weights object is passed to getForwardSignal() as a new `profile`
  parameter. getForwardSignal() computes the highest-weight dimension and includes it in
  the system prompt: "For this site, [dimension] is the highest-priority dimension
  ([weight]% of overall signal weight). Lead your observation with what most impacts
  [dimension]."

ARCHITECTURAL DECISIONS AND RATIONALE
─────────────────────────────────────────────────────────────

WHY profiles.js as a separate module:
One place to change routing logic and calibration values. scorer.js and worker.js import
from profiles.js through the scorer.js re-export chain. No consumer of profiles.js needs
to know which file it lives in — they just import resolveProfile from scorer.js.

WHY seoBaselineGrade overrides the computed grade rather than adjusting the score:
The underlying score is the authoritative measurement. The baseline is a contextual
adjustment — a mid-market firm should not fail their own audit for having a 76 instead
of an 80 when their site is functionally competitive. The override is transparent: the
raw score is still in the response as seoScore.

WHY authorityPenaltyReduced applies at the finding band level, not the score:
The authority score (max 10) feeds into the SEO total. Adjusting it at the band level
leaves the raw score intact for comparison purposes while changing the display severity
that the business owner actually reads.

WHY hookFailureIsCritical adjusts color, not the hook value:
The positioning hook IS missing — that is the observable truth. Changing the card color
from red to yellow acknowledges that the business does not use its homepage as a cold
conversion tool, without misrepresenting what is actually there.

WHY classifyEntity() before the main try block:
The private company path requires DataForSEO and will throw if it fails. The enterprise
path wraps DataForSEO in its own try-catch. Calling classifyEntity() before the shared
try block lets the two paths fail independently without one masking the other.

WHY three sentences for Forward Signal instead of two:
Two sentences left the observation incomplete. The middle sentence (why it matters)
provides the context a business owner needs to evaluate whether to act. The third sentence
turns the observation into a directive. The structure (observe → explain → act) maps to
how an advisor actually delivers a recommendation.

OPEN QUESTIONS
─────────────────────────────────────────────────────────────

1. Classification edge cases: publicly traded micro-cap companies with no consumer
   presence — do they belong in the enterprise path or private company path? Current
   prompt says "publicly traded" which could over-classify.

2. Audience Clarity keyword diversity: the list covers sports/media and B2B terms but
   not healthcare, legal, or education sector vocabulary. Sector-specific terms may be
   needed for enterprise clients from those verticals.

3. Content Depth section path matching: currently matches /news, /blog, /video, /scores,
   /insights, /resources, /learn, /knowledge, /research. Professional services and legal
   firms may have different deep-content path patterns (/practice-areas, /case-studies,
   /attorneys) that would score them low unfairly.

4. Enterprise Forward Signal framing: the current prompt frames the signal as advice for
   "a private company operator studying this brand." If the brand itself ever requests a
   Signal, the framing would need to change.

5. Compare mechanic for enterprise: currently blocked for enterprise path by hiding the
   compare box. If a use case emerges for comparing two known brands, the mechanic would
   need a separate enterprise-vs-enterprise branch.

6. Profile routing for edge cases: a midmarket company with no investors, no existing
   client base, and mixed goals falls to mid_mixed. This is a reasonable default but the
   weights (25 positioning / 45 SEO / 30 agent) skew heavily to SEO, which may not fit
   every midmarket company. Future: add a fourth option in Q2 for "general audience" to
   give these firms a cleaner routing path.
